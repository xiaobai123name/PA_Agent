# 交易二元决策树

本文件按二元判断方式组织交易决策逻辑。每一步只问一个问题：是 / 否。如果答案不清楚，默认进入“等待”，而不是强行做多或做空。

核心顺序：

1. 先判断市场状态。
2. 再判断方向偏好。
3. 再找入场信号。
4. 再检查风险收益（含止损、目标）。
5. **交易者方程（10.3）在已确定止损与目标之后、程序执行解析（§11）之前做最终裁定。**
6. ~~入场后逐K管理。~~（PA Agent 不持仓，跳过 §12–§13）
7. ~~市场状态变化时立刻切换策略。~~（单次分析；请用户重新跑分析）

**阶段划分（与 PA Agent 两阶段分析一致）：**
- 阶段一（诊断闸门 §0–§2）：只做定性判断，**不做**交易者方程数值检验。
- 阶段二（策略与执行 §3–§11、§14）：AI 输出结构分支、入场信号(§9)、风险收益(§10，含 10.3) 和唯一 `entry_intent`；程序随后解析下单方式(§11)。

**四条全局硬禁令（与 `提示词大纲_人设与思维方式.txt` 一致，覆盖下文各专篇）**
1. **禁止逆势**：只顺 `direction` / Always In / 近期主方向；§7 反转、MTR、楔形反转、区间假突破反手等**不得**产出逆势三价。
2. **禁止追高潮**：`climax_risk` 预警/触发或高潮确认后禁追原方向；**禁止 SCS**；仅 SPS 回撤顺势或 wait。
3. **禁止仓位管理**：不输出手数、比例、分批、移止损、跟踪止损、保护利润。
4. **成交量非必要条件**：程序确认可用时只作确认或降级证据；不得单独改变方向、通过闸门、否决交易或生成三价。

---

## 0. 总原则

### 0.1 是否看得懂当前市场？

是：继续下一步。

否：等待。

处理：
- 不开仓。
- 不猜方向。
- 不因为“感觉要动了”而交易。
- 等待更多K线，让结构变清楚。

### 0.2 是否具备继续深入分析的条件？（定性，非交易者方程）

是：允许进入阶段二，继续评估具体策略分支、入场与风控。

否：等待。

本步仅做**定性**判断（结构是否足以支持后续分析），须考虑：
- 当前周期位置是否可识别。
- 长程结构窗口背景是否可描述。
- 价格位置是否极端模糊。

**不在此步**用胜率×回报做数值检验；数值检验见 **§10.3**（止损与目标确定之后）。

### 0.3 交易者方程（原则提醒，非执行节点）

公式：胜率 × 预期回报 > 败率 × 预期风险

本条为总原则：**最终是否允许下单，以 §10.3 的评估为准。**

- 阶段一（诊断）**不要**填写本节点为「是/否」闸门。
- 阶段二在 **§10.1–§10.2 已定止损与目标之后** 必须执行 **§10.3**，通过后才允许程序进入 **§11 下单方式解析**。

## 1. 是否可以决策

### 1.1 数据是否足够？

否：
- K线数量少于20根。
- 数据质量不足。
- 图形太乱无法判断。

处理：
- 输出：无法决定。
- 操作：等待。
- 不加载交易策略。
- 不做任何方向性判断。

是：进入下一步。

### 1.2 是否能识别市场周期？

必须能归入以下状态之一：

- 尖峰 / 极速上涨或极速下跌。
- 微型通道。
- 紧凑通道。
- 常规通道。
- 宽通道 / 台阶。
- 趋势型交易区间。
- 普通交易区间。
- 极端交易区间。

否：
- 输出：unknown。
- 操作：等待。
- 原因：诊断不清，决策必错。

是：进入下一步。

### 1.3 当前市场是否极端混乱？

判断条件：
- 大量K线重叠。
- 上下影线频繁。
- 20EMA水平并与价格纠缠。
- 没有清晰方向。
- 任意入场接近抛硬币。
- 胜率接近50%，扣除滑点手续费后期望为负。

是：
- 状态：极端交易区间。
- 操作：不交易。
- 等待以下任一信号：
  - 连续3到5根强趋势棒突破。
  - 价格形成清晰通道。
  - 重要消息后市场选择方向。

否：进入方向判断。

## 2. 多空方向判断

### 2.1 是否存在明确惯性方向？

判断依据：
- 20EMA是否上倾或下倾。
- 最近20根K线重心是否上移或下移。
- 波段高低点是否有序。
- K线重叠度是否低。
- 是否连续出现同方向趋势棒。
- 价格是否持续运行在EMA一侧。

否：
- 进入区间逻辑。
- 不做趋势跟踪。
- 只考虑区间边界交易，或等待。

是：进入趋势 / 通道 / 尖峰逻辑。

### 2.2 长程大背景方向与近期方向的关系？

提取方法：
- 长程大背景方向：在 **K{n}–K41**（Brooks 亦称全窗口 K{n}–K1，n 通常为 100）的完整结构上判断主要趋势方向
  （多头/空头/区间/中性）
- 近期方向：来自 §2.3（当前方向判断）的结果
- 比较两者关系，分三种情况：

**同向（一致）：**
- 强共振，置信度可提高。
- 顺势交易的可靠性最高。

**反向（冲突）：**
- **不否决近期方向。** Al Brooks："新趋势比旧有趋势更能代表当前的现状，
  也应该成为你交易的核心"（《逐棒分析》第88页）
- 近期方向是当前交易的主方向，大背景方向作为风险提示。
- 策略调整：
  - 不输出任何仓位/资金/手数建议；仅用“信号强弱/风险等级/是否建议交易”表达保守程度。
  - htf_context 中记录大背景方向（如"AIS"），作为磁力位警告。
  - 若近期方向明确（§2.3 非中性）且 §2.5 惯性够强 →
    按近期方向执行，不等待。
  - 若近期信号弱（§2.3 中性 + §2.4 否 + §2.5 否）→ 等待更多K线确认。
  - 不要求"必须出现楔形反转/双重顶底等特殊形态"才允许交易——
    近期尖峰本身就是有效的顺势方向（"新趋势是交易核心"）。

**中性/无明确大背景方向：**
- 中立，按近期信号窗口独立判断。
- 交易决策完全由 §2.3–§2.5 决定。

### 2.3 当前方向是多头还是空头？

多头条件：
- EMA上倾。
- HH + HL结构。
- 收盘重心上移。
- 多头趋势棒占优。
- 回撤小且被买入。

空头条件：
- EMA下倾。
- LL + LH结构。
- 收盘重心下移。
- 空头趋势棒占优。
- 反弹小且被卖出。

如果都不明显：
- 视为中性。
- 进入区间 / 等待逻辑。

### 2.4 当前是否处于 Always In 状态？

Always In Long（AIL）条件：
- 近端 **K8–K1** 多数收在 EMA 上方（加权同侧占比高）。
- EMA上倾，且价格回撤浅。
- HH + HL结构清晰。
- 多头趋势棒占优，空头反击缺少跟随。

⚠️ 20GB特殊规则：连续约20根K线未触及EMA（即"20GB"，定义见「文件16-K线信号识别.txt」），表示趋势极强。在20GB状态下：
  - 绝对不逆势。
  - 首次触及EMA是顺势刮头皮机会（非反转信号）。
  - 两次尝试失败（触及EMA后无顺势跟随）则放弃，不做第三次。
  - 识别到20GB时，detected_patterns 写入 `20gb`。

Always In Short（AIS）条件：
- 近端 **K8–K1** 多数收在 EMA 下方（加权同侧占比高）。
- EMA下倾，且反弹弱。
- LL + LH结构清晰。
- 空头趋势棒占优，多头反击缺少跟随。

⚠️ 20GB空头对称规则：连续约20根K线未触及EMA，首次触及EMA是顺势做空机会。

是：
- AIL：默认只找做多方案，不做首个做空反转信号。
- AIS：默认只找做空方案，不做首个做多反转信号。
- 只有满足趋势线突破 + 前极点测试失败，才允许进入反转**诊断**路由（§7：`reversal_attempt` / `mtr`；**禁止逆势三价**）。

否：
- 视为中性或弱惯性。
- 降低方向置信度，优先等待区间边界、二次入场或更多K线确认。

### 2.5 当前惯性强度是否足够支持趋势跟踪？

强惯性条件：
- K线重叠少。
- 回撤通常不超过1到3根K线。
- 趋势方向K线连续出现，收盘接近方向极点。
- 没有连续两根K线收在EMA另一侧。
- 反向信号缺少跟随。

是：
- 允许趋势跟踪或顺势回撤交易。
- 逆势信号默认视为回撤或旗形，而不是反转。

否：
- **不代表 gate=wait，不在此节点短路阶段二。** 继续进入策略分支（§3–§6），切换为"等待反弹到位后的顺势信号"模式。
- 具体处理：
  - 不做激进趋势跟踪（不追突破、不在反弹中追顺势）。
  - 转为等待反弹衰竭后的入场信号：宽通道等反弹至上边界/EMA附近出现空头信号棒；常规通道等回撤至38%–50%出现顺势信号棒。
  - 在 watch_points 中明确写出触发条件（位置+形态），而不是泛泛说"等待结构明朗"。
- **只有当市场状态完全无法识别（§1.2 unknown）或极端混乱（§1.3 extreme_tr）时，才在阶段一输出 gate=wait。惯性暂时中断不构成 gate=wait 的条件。**

## 3. 是否为尖峰 / 极速行情

### 3.1 是否满足尖峰条件？

⚠️ 以下分级定义与「市场诊断框架.txt」状态1中【统一 Spike 分级定义】完全一致，以该定义为准。

做多尖峰条件分级：
- 弱尖峰候选（1根）：恰好1根超大突破棒 + 下一根有跟随 → 只记录 `spike_candidate`，**1根时 cycle_position 不出 spike**；主状态输出 `micro_channel` / `tight_channel` / `trending_tr`（择最近似），**gate_result 仍为 proceed**；watch_points 写明等第二棒确认；**禁止追单**。
- 最低可路由尖峰（2根）：2根以上连续大型多头趋势棒，实体重叠 < 30%，收盘持续创新高。**仅允许 SPS（等回撤后顺势入场），禁止 SCS 追单。**
- 标准尖峰（3~5根）：3到5根，实体重叠 < 20%，几乎无上尾线。**仅 SPS / 尖峰旗形顺向突破，禁止 SCS。**
- 强尖峰/高潮风险（6根以上，或任意序列+衰竭信号）：按高潮判断（见市场诊断框架状态1），**禁追**，仅 wait 或等回撤后 SPS

做空尖峰条件分级（对称）：
- 弱尖峰候选：1根超大突破棒 + 下一根有跟随 → 同上，禁止追单
- 最低可路由：2根以上，实体重叠 < 30%，收盘持续创新低 → **仅 SPS，禁止 SCS**
- 标准尖峰：3~5根，实体重叠 < 20% → **仅 SPS / 尖峰旗形顺向突破，禁止 SCS**
- 强尖峰/高潮风险：同多头对称处理

否：进入通道或区间判断。

是：继续。

### 3.2 尖峰是否仍在进行中？

是：
- 只顺势，不反转。
- 上涨尖峰：只做多。
- 下跌尖峰：只做空。
- 禁止因为“涨太多 / 跌太多”逆势。

否：
- 进入尖峰结束后的三路径判断。

### 3.3 尖峰进行中是否允许追？（SCS 已废止）

**否（硬规则，无例外）：**
- **禁止 SCS / 尖峰追单 / 市价追突破。**
- 尖峰进行中只观察；入场仅经 §3.4 **SPS**（回撤后顺势信号）或 §3.4 **尖峰旗形顺向突破**（非追高潮棒；≠ 文件24 失败 FF 逆势 FFES）。
- `climax_risk=warning/triggered` 时本节必答「否」。

### 3.4 尖峰是否出现第一次暂停或回撤？

否：
- 继续观察。
- 不逆势。
- 不新增反向交易计划。

是：
- 如果回撤只有2到4根K线，且出现顺势信号棒：
  - 使用 SPS 尖峰回撤入场。
  - 顺势入场。
- 如果形成第一次旗形：
  - 使用**尖峰旗形顺向突破**入场（SPS 变体，顺主趋势）。
  - 等旗形突破后顺势入场；**禁止**追高潮棒。

### 3.5 尖峰后进入哪条路径？

#### 路径A：回撤小于38.2%到50%

判断：
- 趋势大概率恢复为通道。
- 概率约60%。

处理：
- 从尖峰策略切换为通道策略。
- 不再盲目追。
- 等回撤到EMA、趋势线、通道边界附近顺势入场。
- 不输出任何仓位建议；仅在结构清晰且交易者方程通过时给出三价方案。

#### 路径B：回撤超过61.8%，并开始横盘

判断：
- 尖峰失败，进入交易区间。
- 概率约30%。

处理：
- 停止趋势跟踪。
- **仅顺 direction 一侧**边界等待（neutral 则 wait）。
- 不追突破；中间不交易。

#### 路径C：趋势线突破 + 前极点测试失败

判断：
- 可能反转（约10%），但**本 Agent 禁止逆势下单**。

处理：
- 仅写入 `reversal_attempt` / `mtr` 诊断与 `watch_points`。
- **不输出**逆原趋势的三价方案；`order_type=不下单`，`terminal=wait`。

### 3.6 是否出现高潮？

⚠️ 高潮判定统一规则（见市场诊断框架状态1）：任意连续强趋势棒序列 + 出现以下任一衰竭信号即触发：
- 长尾线（尾线 > 实体长度50%）
- 小实体棒（实体 < 近期均值30%）
- 反向趋势棒收盘

买进高潮触发（不依赖固定根数）：
- 连续上涨趋势棒序列 + 出现上述任一衰竭信号
- 特别警示：6根以上连续多头趋势棒时，即使尚未出现衰竭，也进入高潮预警状态

卖出高潮触发（对称）：
- 连续下跌趋势棒序列 + 出现下述任一衰竭信号（长下影、小实体、多头趋势棒）
- 6根以上连续空头趋势棒时进入高潮预警

是：
- **禁止追原方向**（含市价/突破追单）。
- 仅 wait，或在 §3.4 回撤后出现**顺主方向** SPS 时再评估。
- 等至少10根K线或更清晰的回撤结构。

否：继续按尖峰或通道策略处理（仍禁止 SCS）。

### 3.7 是否为第二次或第三次高潮？

第二次高潮：
- 不追原方向。
- 等回撤后再考虑**顺主方向**新入场（SPS），否则 wait。

第三次高潮：
- 高概率接近趋势末端。
- 不追原方向；**禁止逆势反转单**；仅 wait 或顺主方向 SPS。

## 4. 是否为通道

### 4.1 是否出现有序波段结构？

上涨通道：
- 高点逐步抬高：HH。
- 低点逐步抬高：HL。
- 至少两组 HH + HL 进入**通道候选**判定（见统一判定树；**仅 2 组时主状态多为 trending_tr**；≥3 组且可画稳定通道线才确认 normal/broad/tight 通道）。

下跌通道：
- 低点逐步降低：LL。
- 高点逐步降低：LH。
- 至少两组 LL + LH 进入**通道候选**判定（同上：2 组≠已确认通道，须核对 cycle_position）。

否：
- 进入区间判断。

是：
- 继续。

### 4.2 通道方向是上涨还是下跌？

上涨通道：
- 核心认知：上涨通道 = 空头旗形。
- 顺势做多仍优先。
- 但通道越久，越要警惕向下突破。

下跌通道：
- 核心认知：下跌通道 = 多头旗形。
- 顺势做空仍优先。
- 但通道越久，越要警惕向上突破。

## 5. 通道宽度判断

### 5.1 是否为微型通道？

条件：
- 2到10根K线。
- 几乎没有回撤。
- 每根K线高低点连续同向推进。
- 常是尖峰的一部分。
- **且未满足尖峰最低路由标准**（见市场诊断框架「spike vs micro_channel 裁决」；满足尖峰标准时优先 spike，不得标 micro_channel）。

是：
- 不逆势。
- 不追第一个突破。
- 第一个突破尝试通常失败。
- 等突破失败后顺势入场。

处理：
- 上涨微型通道：向下突破失败后做多。
- 下跌微型通道：向上突破失败后做空。
- 止损：突破极点外1跳动。
- 目标：微型通道高度或尖峰测量移动。

否：继续。

### 5.2 是否为紧凑 / 窄通道？

⚠️ 通道分类统一以回撤幅度为主维度，斜率、绝对跳动数和视觉宽窄只作辅助描述，避免双维度判断结果不一致。

条件（全部满足）：
- 回撤 < 30%（量化，含1-3跳动微回撤）。
- K线重叠少（平均重叠 < 20%）。
- 趋势棒比例 > 65%。
- EMA紧贴价格，两者间无连续2根K线完全在EMA另一侧。
- （辅助描述）斜率大、几乎垂直于水平基线。

是：
- **只做顺势**（顺 `direction` / Always In）。
- **禁止逆势**刮头皮或通道边界反手。
- 不追通道边界突破。
- 等回撤到EMA或趋势线附近入场。

上涨窄通道：
- 首选：High1 / High2 回撤买入（仅当 direction=bullish 或 AIL）。
- 止损：信号棒低点外1跳动。
- 目标：最近结构位、通道上轨或测量移动候选。

下跌窄通道：
- 首选：Low1 / Low2 反弹卖出（仅当 direction=bearish 或 AIS）。
- 止损：信号棒高点外1跳动。
- 目标：最近结构位、通道下轨或测量移动候选。

否：继续。

### 5.3 是否为常规通道？

条件：
- 回撤30%到50%（量化）。
- 波段清晰，每波段至少有2-3根信号K线。
- （辅助描述）通道斜率适中。
- 每个波段有交易空间。

是：
- **只做顺势**；禁止逆势、边界反手、MTR 反转单。
- 顺势入场：回撤到38.2%到50%区域，出现信号棒。
- 止损：信号棒极点外1到2跳动。
- 顺势目标：通道对边或测量移动候选。

否：继续。

### 5.4 是否为宽通道 / 台阶？

条件（全部满足）：
- 回撤深，50%到78.6%（量化，主维度）。
- 波段大，每波段通常超过10根K线。
- K线重叠 > 40%。
- 突破后常有突破测试。
- （辅助描述）斜率小（< 30°）。

⚠️ 与 trending_tr 边界模糊时：若已有3组有效 HH+HL / LL+LH 且可画稳定通道线，输出 broad_channel；若只有2组或通道线不稳定，输出 trending_tr。不得仅因回撤45%-55%或均线微倾斜就把2组结构升级为 broad_channel。

是：
- **仅顺主方向**（顺 `direction` / Always In）；**禁止**通道对侧边界逆势刮头皮。
- 核心模式：突破后不追，等突破测试后**顺主方向**入场。

上涨宽通道（direction=bullish）：
- 下边界/回撤做多；上边界**不做空**。
- 上破通道线不追，等回踩做多。

下跌宽通道（direction=bearish）：
- 上边界/反弹做空；下边界**不做多**。
- 下破通道线不追，等反弹测试做空。

direction=neutral：宽通道内默认 wait，除非 §9.0P 仅有**单侧**顺局部结构且与 §2.4 不冲突的方案。

风控：
- 止损：信号棒极点外1跳；边界噪声大时扩至最近波段极点外1跳。
- 目标：通道中线、对侧边界、或下一台阶候选。

否：继续评估是否为区间。

### 5.5 是否为收缩台阶？

条件：
- 至少三段推进。
- 每段推进幅度递减。
- 动能衰退。
- 趋势线多次重画并变平。

是：
- trade_confidence 下调；不在最后一推追顺势。
- **禁止逆势反转单**；等回撤后顺主方向 SPS 或 wait。

否：
- 继续按当前通道类型执行。

## 6. 是否为交易区间

### 6.1 是否存在清晰上下边界？

⚠️ 先确认已通过「市场诊断框架.txt」中的【统一判定树】走到区间态。满足以下全部条件才进入区间策略：
- 无满足3组的 HH+HL 或 LL+LH 序列（否则走通道策略）
- 上下边界至少各测试2次
- EMA 水平或斜率 ≤ 15°（与 `市场诊断框架.txt` trending_tr 一致）
- 大量K线重叠

否：
- 重新评估是否为 trending_tr 或 broad_channel。

是：
- 进入区间策略。

### 6.2 是普通交易区间还是趋势型交易区间？

普通交易区间：
- 无明显方向偏好。
- 上下边界对称。
- 均线水平。
- 突破80%失败。

处理：
- direction=bullish：仅下边界/回撤**做多**。
- direction=bearish：仅上边界/反弹**做空**。
- direction=neutral：**不下单**（禁止双边刮头皮）。
- 中间位置不交易；不追突破。

趋势型交易区间：
- 有轻微方向偏好；均线可微倾。
- **仅顺 direction 一侧**边界/回撤入场；禁止逆势。
- 顺势目标可取区间宽度 60%–80% 结构候选；须 §10.3。

### 6.3 当前价格是否在区间边界？

否，在中间1/3：
- 不交易。
- 等价格到边界。

是，在下边界：
- direction=bullish 或 neutral+§2.4 AIL：等看涨信号，可做多。
- direction=bearish：**不做多**（逆势），wait。

是，在上边界：
- direction=bearish 或 neutral+§2.4 AIS：等看跌信号，可做空。
- direction=bullish：**不做空**（逆势），wait。

### 6.4 是否出现Barbwire / 极窄区间？

⚠️ Barbwire 权威标准：紧凑度 **<25%**（见文件21）；下列「区间宽度 < 平均波段30%」仅为**辅助描述**，不得覆盖 25% 裁决。
- EMA水平并纠缠。
- 方向完全不明。

是：
- 不在里面交易。
- 等尖峰级别突破。
- 突破后仍不追，等回踩测试成功再顺势入场。

否：
- 继续顺 direction 边界等待或突破测试评估。

### 6.5 区间是否出现突破？

否：
- 继续顺 direction 边界等待（neutral 则 wait）。

是：
- 判断突破质量。

高质量真突破：
- 连续3到5根同方向强趋势棒（**不依赖成交量**）。
- 实体大，影线短；突破后未快速回原结构；EMA跟随。
- 最好有回踩测试成功。

处理：
- **禁止追第一根突破**（高潮/尖峰棒禁追）。
- 等回踩测试后**仅当突破方向与 direction 一致**时顺势入场。

低质量假突破：
- 1–2根、影线长、无跟随、快速回区间。

处理：
- **禁止反手逆势单**。
- 若假突破方向逆主方向：wait 或评估顺主方向的 §9.0P（非追突破）。
- 写入 `breakout_failure`；`order_type=不下单` 除非有顺主方向结构位方案。

### 6.6 是否为三角形形态？

⚠️ 三角形已在 pattern_enum.md 中定义（ascending_triangle / descending_triangle / symmetrical_triangle / expanding_triangle），detected_patterns 必须写入对应枚举值。

三角形是交易区间中的特殊收敛形态，本质仍是区间，但有独特的交易规则。

**上升三角形**（上边界水平 + 低点渐高）：
- 向上突破 + direction=bullish → 回踩测试成功后**做多**
- 向上突破 + direction≠bullish → wait，仅诊断
- 向下跌破 + direction=bearish → 回踩测试成功后**做空**（须与 direction 一致）
- 向下跌破 + direction≠bearish → 仅诊断 `reversal_attempt`，**禁止逆势三价**

**下降三角形**（下边界水平 + 高点渐低）：
- 向下突破 + direction=bearish → 回踩测试成功后**做空**
- 向下突破 + direction≠bearish → wait，仅诊断
- 向上突破 + direction=bullish → 回踩测试成功后**做多**（须与 direction 一致）
- 向上突破 + direction≠bullish → 仅诊断 `reversal_attempt`，**禁止逆势三价**

**对称三角形**（边界收敛）：
- 等收盘突破 + 跟随；**仅顺突破方向且与 direction 一致**时评估入场

**扩张三角形**（边界向外扩散）：
- **不交易**。这是最混乱的形态，边界不可靠。
- 等待形态结束、方向明确后再评估。

风控（统一）：
- 止损：信号棒极点外1跳
- 目标：突破方向测量移动（区间高度投影）
- 不输出任何仓位/资金/手数建议
- 突破不追，等回踩测试成功后再入场（与 §6.5 突破测试原则一致）

---

## 7. 是否出现反转（仅诊断，禁止逆势下单）

⚠️ 本节只用于 `detected_patterns` / `watch_points`；**不得**因 §7 产出逆原趋势的三价方案。

### 7.1 是否只有单根反转K线？

是：
- 不够。
- 在趋势中，单根反转K线大多只是回撤。
- 等更多确认。

否：
- 继续检查。

### 7.2 是否同时满足两大反转条件？

条件一：
- 趋势线被收盘价有效突破。

条件二：
- 前极点测试失败。
- 上涨趋势：无法再创新高。
- 下跌趋势：无法再创新低。

否：
- 不做反转。
- 视为旗形或回撤。
- 继续按原趋势处理。

是：
- 仅提高反转**诊断**置信度；**禁止逆势入场**。
- 继续顺主方向策略或 wait。

### 7.3 是否出现二次入场？

（顺势二次入场 H2/L2、突破回踩等仍可用；**逆主趋势的 MTR/双顶底反手禁止下单**。）

二次入场类型（顺主方向）：
- 趋势回撤 H2/L2、突破回踩、区间边界第二次测试（须与 direction 一致）。

否：
- 等待。

是（且方向与 direction / Always In 一致）：
- 可评估入场；止损：信号棒极点外1跳动；须 §10.3。

是（但方向逆主趋势）：
- **不下单**；`terminal=wait`。

### 7.4 二次入场是否失败？

是：
- 不做第三次尝试（**末端楔形三推**场景亦禁止追顺势、禁止逆势反手，仅 wait）。
- 重新评估结构；很可能原趋势仍未结束。

否：
- 继续顺主方向管理（单次方案层面）。

---

## 8. 是否出现楔形 / 三推

### 8.1 是否有三推结构？

条件：
- 同方向推进三次。
- 每次推进动能递减。
- 趋势线与通道线收敛。
- 第三推明显弱于前两推。

否：
- 忽略。
- 不强行画楔形。

是：
- 判断楔形性质。

### 8.2 楔形是回撤还是反转？

楔形回撤：
- 楔形方向与主趋势相反。
- 通常10到20根K线。
- 出现在趋势中的调整阶段。

处理：
- 顺主趋势交易。
- 上涨趋势中的下降楔形：向上突破做多。
- 下跌趋势中的上升楔形：向下突破做空。
- 目标：前极点或测量移动。

楔形反转：
- 楔形方向与主趋势相同；趋势末端；动能衰竭。
- 意味反转*可能*发生，但多数仍失败变旗形或区间。

处理：
- **仅诊断** `reversal_attempt` / `wedge`。
- **禁止追顺势**（含 H1/H2 追原方向、SCS、追突破）。
- **禁止逆势下单**；`order_type=不下单`；`watch_points` 写突破/回撤监测与失效条件。

### 8.3 是否为抛物线楔形？

条件：
- 价格加速推进；每一推斜率更陡；出现高潮感。

是：
- **禁止追趋势、禁止追高潮**。
- 仅 wait 或顺主方向深回撤后的 SPS。

否：
- 按楔形回撤处理（顺主趋势）；楔形反转仍禁止下单。

## 9. 入场信号二元检查

任何交易入场前，都必须逐项检查。

### 9.0 信号棒是否已经收盘且质量足够？

**本节点只评估「当前是否已有合格信号棒」**（实体清晰、方向一致、可挂止损于棒极点）。与计划型限价无关。

否：
- 检查是否存在明确的 pullback 或 breakout 意图；不存在就使用 none。
- 不得因为缺少信号棒而默认生成限价单。
- 未收盘的K线只作为观察对象，不作为最终信号棒。

是：
- 继续 §9.1–§9.7 检查信号棒方向、长度、背景与跟随。

### 9.0P 结构回撤意图评估（仅 entry_intent=pullback 时填写）

本节点只评估是否确实需要等待价格回到结构位。它不是缺少信号棒时的默认第二路径。

⚠️ **与二次入场价陷阱规则的分界**（文件15 §6）：
- **§9.0P 边界/回撤计划型限价**：在支撑/阻力/通道边界/EMA 挂更优价属正常，**不受**「二次入场价更优=陷阱」约束。
- **H1/H2/L2 计数二次入场**：第二次价格明显优于第一次 → 仍按陷阱规则警惕，优先 wait 或重判结构。
- **§9.0P 仅允许顺 direction / Always In 一侧**；禁止逆势边界限价。

否：
- 无计划型 setup：区间/通道 **中部**、barbwire 且无边界锚点、无法给出结构失效 stop 与有效 target、**order_direction 逆 direction**、§14 触犯。
- terminal.outcome=wait，node_id=**9.0P**（或 9.0，若未写 9.0P）。

是：
- 阶段一 gate_result=proceed，且交易计划明确要求价格回到 **支撑/阻力/通道边界/EMA/顺势回撤位**。
- **依据是周期 + 方向背景 + 结构位**，不要求独立信号棒。
- `signal_bar.bar=null`，`quality=invalid/weak`，`pattern=none`；`entry_bar` 设 not_triggered/pending。
- 设置 `entry_intent=pullback`、`order_type=限价单` 并继续 §10；若实时价格关系不成立，程序直接拒绝，不会改成其他类型。

**§9.0P=是 时的 K1 对照（仅计划型限价）：**
- 只检查 **entry 是否在 K1.close 的正确一侧**（做多限价 entry < K1.close；做空限价 entry > K1.close）。
- **禁止**因 K1 影线曾触及 entry 就判失效——限价单是等**未来**回撤/反弹，不是要求 K1 未路过。
- K1 已触及 **stop** 仍判方案无效。

**§9.0P 与 direction=neutral（强制）：**
- 阶段一 `direction=neutral` 时，**禁止**在区间/通道上下边界同时给出仅顺 direction 一侧方案。
- 须读取阶段一 `gate_trace` §2.4：
  - **AIL** → 仅允许**做多侧** §9.0P（下边界/支撑/回踩做多）；**禁止**阻力做空。
  - **AIS** → 仅允许**做空侧** §9.0P（上边界/阻力/反弹做空）；**禁止**支撑做多。
  - §2.4 答案为「否」（非 Always In）→ §9.0P 默认 **wait**；仅 §9.0=是 且有合格信号棒时可下单。
- **direction=neutral 时**：§9.0P 默认 **wait**；禁止双边边界限价。

**§9.0P 方案连续性（强制，配合程序「上一轮方案」块）：**
- 若程序标注上一轮方案**未失效**：默认 `order_type=不下单`、`terminal.outcome=wait`，在 watch_points 说明仍等待上一轮触发。
- **同结构位反手冷却**：若新 entry 与上一轮 entry 相差≤3跳，在 **3 根**已收盘 K 线内**禁止反向**新方案；仅当 K1 **收盘**触及/突破上一轮止损或 `invalidation_condition` 结构位（程序标注「已失效」）后才可反手。
- 覆盖连续性规则时，须在 `decision.reasoning` **首句**写明「连续性覆盖」及 K 线收盘证据。

**§10 定价顺序（计划型限价推荐）：**
1. 定 **entry**（结构位/边界/回撤位）
2. 确定唯一的**结构失效位**，加明确的 tick 或 ATR 缓冲，定 **stop_loss_price**；止损不由 RR 反推
3. 定 **take_profit_price（TP1）**（最近有效结构目标）
4. 定 **take_profit_price_2（TP2）**（更远结构目标：MM、通道对边远端、区间翻测等；**必填**）
5. 按三价计算 RR=回报÷风险；若 RR **> 1.5**，必须填写 `high_rr_review` 复核结构止损、TP1 与胜率依据；复核通过则允许原价下单，否则拒绝，**不修改 stop 或 TP**
6. 若 RR **< 1.0** 或交易者方程不通过，重新评估整套结构方案或 reject；**禁止**只为满足 RR 收紧/放宽 stop、缩小 TP 或移动 entry
7. 结构止损距离过大时等待更小结构信号或放弃；本 Agent 不输出仓位，执行层调整仓位时必须保留结构止损
8. 几何校验：做多 stop<entry<TP1<TP2；做空 TP2<TP1<entry<stop

高质量信号棒：
- 与计划方向一致。
- 实体清晰，影线较短。
- 收盘接近计划方向极点。
- 出现在正确背景中：趋势回撤末端、区间边界、突破回踩、楔形或反转双确认之后。

低质量信号棒：
- 十字星。
- 长外包棒。
- 与前几根K线大幅重叠。
- 出现在区间中部或铁丝网中。
- 过长导致止损过大。

**低质量信号棒的处理（等待触发条件）：**
当前K线（K1）为低质量信号棒（如十字星、doji、外包棒）时，不直接放弃，而是在 watch_points 中明确下一根K线（K0）的触发条件：
- 宽通道/常规通道空头背景：若 K0 为空头趋势棒且收盘低于 K1 低点 → K0 成为新的信号棒，**止损改为 K0 高点上方1跳**，以 K0 为基准重新通过 §10.1–§10.3 评估。
- 宽通道/常规通道多头背景：若 K0 为多头趋势棒且收盘高于 K1 高点 → K0 成为新的信号棒，**止损改为 K0 低点下方1跳**，以 K0 为基准重新通过 §10.1–§10.3 评估。
- 紧凑通道空头：若 K0 为空头趋势棒（Low 1 或 Low 2 形态）→ K0 高点外1跳为止损，重新评估方程。
- **关键原则：K0 触发后的止损与交易者方程，必须以 K0 的极点为基准重新计算，而不是沿用 K1 的极点。** K0 通常比 K1 更短，止损更小，方程更容易通过。
- 当前分析轮次没有 K0，故输出 wait；**应在 watch_points 中明确写出 K0 触发条件以及触发后的止损逻辑**，而不是仅说"等待"。
- 如果 K0 不满足上述条件，继续等待，不强行入场。

### 9.1 信号K线是否已经收盘？

否：
- 等待。
- 不在K线形成过程中提前进场。

是：
- 继续。

### 9.2 信号K线方向是否与计划方向一致？

做多要求：
- 阳线。
- 收盘接近高点。
- 下影线较短。
- 最好在支撑、EMA、趋势线或区间下边界。

做空要求：
- 阴线。
- 收盘接近低点。
- 上影线较短。
- 最好在阻力、EMA、趋势线或区间上边界。

否：
- 降低 trade_confidence 或放弃。

是：
- 继续。

### 9.3 信号棒是否过长？

是：
- 风险可能过大。
- §10.2 应判「是」→ 等待更小信号棒或放弃；**禁止**用固定跳数/资金管理阈值替代结构止损价位。

否：
- 使用信号棒极点止损。

### 9.4 是否是第一次入场？

是：
- 趋势顺势交易可以接受。
- 逆势/MTR 反转架构：仅诊断 + 等待 H2/L2 监测条件（**禁止逆势三价**，见 §7）。

否，是第二次入场：
- 胜率更高。
- 优先考虑。

### 9.5 是否有跟随？

有：
- 提高 trade_confidence；结构目标（TP1/TP2）可取更远档。**PA Agent 不管理持仓**，禁止写「持有/加大目标/加仓」。

无：
- 信号可能失败。
- 降低 trade_confidence，偏向等待确认或放弃；不输出减仓/收紧止损/退出等持仓管理指令。

### 9.6 是否为二次入场？

是：
- 胜率通常高于第一次入场。
- 反转、楔形、趋势线突破回测、区间边界第二次测试，应优先等待二次入场。
- 若二次入场价格与第一次相同或略差，说明市场要求为更多确认支付成本，结构更健康。

否：
- 趋势顺势交易可以接受第一次入场。
- 逆势反转、弱背景、交易区间中部附近，不做第一次入场，继续等待。

警惕：
- 如果二次入场价格明显优于第一次，可能是陷阱或原结构误判。
- 二次入场失败后，不做第三次尝试（**末端楔形三推**亦仅 wait，禁止追顺势/逆势）。

### 9.7 入场棒是否强势且有跟随？

⚠️ **计划型限价（§9.0P=是）**：尚无入场棒 → 本节点记「否」或 N/A，reason 写明「等待触发，尚无入场棒」；**不**因此否决方案，继续 §10。

是（信号棒/止损单路径）：
- 入场棒收盘方向与计划一致。
- 收盘接近方向极点。
- 后续1到2根K线继续同方向推进或至少不明显反向。
- 可以继续进入风险收益检查。

否：
- 信号可能失败。
- 降低 trade_confidence，偏向等待确认或放弃；不输出收紧止损等持仓管理指令。
- 若突破后无跟随，优先按假突破或失败信号处理。

## 10. 风险收益二元检查

### 10.1 是否能明确止损？

否：
- 不交易。

是：
- 继续。

【⚠️ 统一止损原则（全系统唯一标准）】

止损位置永远基于结构失效位，不基于固定跳数。固定跳数（如"边界外1~3跳"）仅用作风险过滤器——判断止损距离是否过大而应放弃，而不是止损本身的设置依据。

**通用规则：止损 = 信号棒极点外1跳（做多：信号棒低点下方1跳；做空：信号棒高点上方1跳）**

各市场状态的止损位置：

| 状态 | 止损设置位置 | 过滤器（超过则考虑放弃） |
|------|------------|------------------------|
| 尖峰回撤（SPS入场） | 回撤极点外1跳 | 止损 > 尖峰高度50% |
| 普通趋势/通道顺势 | 信号棒极点外1跳 | 止损 > 通道宽度40% 或 > 2倍ATR |
| 普通区间边界 | 信号棒极点外1跳，边界作为结构参考 | 止损 > 区间宽度10% |
| 宽通道（边界噪声大） | 信号棒极点外1跳；边界刺穿频繁时扩至波段极点外1跳 | 止损 > 通道宽度50% |
| 铁丝网/Barbwire | 不做，或等待突破后以突破棒极点外1跳止损 | 边界外1~2跳无效，最少需3~5跳缓冲才考虑 |
| 反转/MTR（仅诊断） | **本 Agent 不产出逆势三价**；若误入 §10 路径 → 直接 `order_type=不下单`，跳过本节 | — |

风险距离过滤阈值（8跳动/信号棒60%高度）的角色：
- 仅作为"止损是否过大"的判断阈值
- 若图表结构止损位置不在此范围内，不得用固定跳数替代图表止损
- 若信号棒过长导致结构止损过大 → 等更小信号，或放弃交易

### 10.2 止损是否过大？

以下任一成立即视为过大（放弃交易或等更小信号）：
- 止损 > 止盈目标距离（盈亏比 < 1:1）
- 止损 > 通道宽度40%（通道态）
- 止损 > 区间宽度10%（区间态）
- 止损 > 2倍ATR
- 风险距离过滤阈值（8跳动/信号棒60%高度）只是辅助参考，不是止损价位：若结构止损大于此阈值，优先等待更小信号；不得用固定跳数替代图表结构止损

是：
- 放弃。
- 或等待更小信号。
- 不为了进场而扩大止损。

否：
- 继续。

### 10.3 交易者方程是否通过？

否：
- 不交易。

是：
- 允许下单。

- 基本要求：
- 刮头皮 / 波段 / 趋势：**TP1** 以最近有结构支撑的价位为准；程序要求 RR≥1.0，交易者方程也须通过。RR>1.5 时 high_rr_review 必须确认结构止损、TP1 与胜率依据，依据成立即可下单，不改写结构止损。
- **TP2** 为更远结构目标（MM 等），必填，不参与 §10.3。
- **反转 / MTR**：仅诊断，**不进入 §10.3 下单路径**；`order_type=不下单`。
- 数值检验：胜率%×回报 > (100−胜率)%×风险（回报、风险**仅来自 TP1**）才允许下单。

**结构目标与止损校验（全场景）：**
- 先按结构定 entry、唯一结构失效 stop（含明确缓冲）、TP1、TP2，再计算 RR。
- 若 RR > 1.5：要求复核结构目标与交易背景；`high_rr_review` 通过则允许原价下单，缺少结构止损、TP1 或胜率依据才拒绝，**不改写 stop 或 TP**。
- 若 RR < 1.0：不得只为通过方程收紧或放宽 stop、缩小 TP1 或移动 entry；重新评估整套结构，仍不满足就 reject。
- **禁止**用 RR 生成没有结构依据的 stop；止损距离过大时等待、放弃，或由执行层在不改变 stop 的前提下另算仓位，本 Agent 不输出仓位。
- **禁止**缩小 TP1 来迎合 RR；结构目标有磁力效应。
- 尖峰 leg 的 MM 常作 **TP2** 候选；TP1 仍取近端结构位。

## 11. 下单方式解析

§11 由程序根据 `entry_intent`、实时形成中 K 线报价、报价年龄和 entry 关系确定。AI 不输出 §11 trace，不得提交 §11 override。

### 11.1 即时入场执行条件是否成立？

仅适用于 `entry_intent=immediate` / `order_type=市价单`：
- 报价必须来自当前品种、当前周期的形成中 K 线，且未超过配置的新鲜度上限。
- 当前价与计划 `entry_price` 的偏差不得超过程序计算的 slippage 上限。
- 超限表示即时机会已经错过，本次方案直接拒绝；不得改成限价单或突破单。

### 11.2 突破入场执行条件是否成立？

仅适用于 `entry_intent=breakout` / `order_type=突破单`：
- 做多触发价必须仍在当前价上方至少一个 tick；做空触发价必须仍在当前价下方至少一个 tick。
- `entry_price` 必须等于 `entry_basis_bar` 对应方向极值外一个 tick。
- 触发位已越过、依据 K 线不存在、极值方向错误或价格不精确时直接拒绝；程序不重算价格，不转成其他订单。

### 11.3 回撤入场执行条件是否成立？

仅适用于 `entry_intent=pullback` / `order_type=限价单`：
- 做多限价必须仍在当前价下方至少一个 tick；做空限价必须仍在当前价上方至少一个 tick。
- 回撤价位于错误一侧或已经不再是等待中的回撤时直接拒绝；不得改成市价追单或突破单。

`entry_intent=none` 不生成 §11 节点。任何 `entry_intent` 与 `order_type` 声明不一致的提案直接生成执行拒绝。

## 12–13. 持仓后管理（PA Agent 不适用）

本程序仅输出**单次**阶段二 JSON（是否给一笔入场方案），不跟踪开仓后 K 线，不做任何持仓后管理（减仓/加仓/移止损/跟踪止损/状态切换调仓等）。

- **禁止**在 decision / watch_points 中写分批出场、移止损到盈亏平衡、持仓后逐 K 管理等指令。
- 若需表达「结构若失效则方案作废」，只写 `invalidation_condition` 与 `watch_points`（针对**尚未成交或重新分析**的条件），而非持仓操作。

## 14. 禁止行为清单

以下行为出现任一项，直接停止交易或放弃当前信号：

- 未识别周期位置就交易。
- 数据不足仍然下单。
- 在极端交易区间中交易。
- 在尖峰中做反转。
- 在买进/卖出高潮后追原方向（含 SCS、市价追尖峰）。
- 使用 SCS 或任何「尖峰追单」路径。
- 在交易区间中追普通突破。
- 在区间中部入场。
- 在趋势型交易区间（trending_tr）中按通道/尖峰逻辑做**中部趋势跟踪或追突破**。
- 在宽通道中追突破。
- 在常规/宽通道中做**任何逆势**（含边界刮头皮、MTR 反手、双顶底反手）。
- 在 Always In Long 中做空；在 Always In Short 中做多。
- `order_direction` 与 `direction` / Always In **不一致**（无 §2.3 合法重判且仍逆主趋势）。
- 因 §7 反转 / `mtr` / 楔形反转产出逆势三价。
- 用当前输入周期长外包棒两端突破直接入场。
- 二次入场价格明显优于第一次仍强行交易。
- 二次入场失败后继续尝试第三次（**末端楔形三推**亦禁止，仅 wait）。
- 信号棒过长仍强行入场。
- 止损距离过大仍强行入场。
- 交易者方程不通过仍下单。
- 为了“看起来更容易成交/更好看盈亏比”而扭曲结构止损（扩大止损或贴噪音）以凑方程。
- 止损后立即反手。
- 连续两笔止损后仍不重新评估市场。

## 15. 最终执行口诀

1. 看不懂，等。
2. 极端混乱，等。
3. 尖峰中，只顺势；**禁止 SCS、禁止追高潮**。
4. 通道/区间，**只顺 direction**；禁止逆势、禁止双边刮头皮。
5. 宽通道，不追突破；仅顺主方向边界/回撤。
6. 区间中，仅顺 direction 一侧；neutral 则 wait。
7. 反转/MTR **只诊断不下单**。
8. 楔形回撤可顺势；**末端楔形/三推/楔形反转：禁止追顺势、禁止逆势下单**。
9. 信号必须等收盘。
10. 止损和交易者方程不过关，直接放弃。
