【文件定位】本文件职责：在已确认上涨通道中，给出**顺势做多**入场方案。仅在诊断结果为上涨通道且 direction=bullish（或 AIL）时加载。

⚠️ **PA Agent 硬约束**：**禁止逆势做空**、禁止追高潮、禁止仓位管理、**不依赖成交量**。

【核心知识】

一、上涨通道交易的核心原则

1. 顺势原则
在上涨通道中，**只做多**（回撤、旗形、突破测试失败后的顺势）。

2. **禁止逆势**
- 禁止通道顶部做空、空头旗形做空、MTR 反手。
- `reversal_attempt` / `mtr` 仅诊断，不下单。

3. 通道类型
- 窄/常规/宽通道：**均只顺势做多**。
- 微型通道：等向下假突破失败后做多，或浅回撤 H1/H2。

二、顺势交易策略详解

策略一：均线回撤买进（Moving Average Pullback）—— High 1 / High 2

这是上涨通道中最可靠的顺势入场模式。

⚠️ High 1/High 2 的精确定义以「文件16-K线信号识别.txt」「文件19-H1H2-L1L2计数.txt」为准。

High 1 模式：
- 价格从均线（通常为20周期EMA）向上运行后，首次回撤至均线附近
- 回撤过程中形成至少一根多头信号棒（阳线或反转K线）
- 在信号棒高点上方1跳动处设置止损买入单（Stop Order）
- 或在信号棒低点下方1跳动处设置保护性止损

High 2 模式：
- High 1之后，价格再次上涨，然后第二次回撤至均线
- High 2的回撤幅度通常比High 1更大，信号也更清晰
- 入场方式和止损设置与High 1相同
- High 2在宽通道中比High 1更常见

关键要点：
- 均线（EMA 20）是动态支撑位，在上涨通道中反复发挥作用
- 回撤至均线时的K线形态至关重要：长下影线、阳线吞没都是强信号
- 如果回撤幅度过大（跌破前一个HL），则High模式失效，不应入场

策略二：旗形突破买进（Flag Breakout Buy）

旗形是通道内最典型的持续形态。

识别旗形：
- 一段快速上涨后，出现3~10根K线的横盘或小幅回撤
- 旗形的上下边界清晰，类似于一个小型交易区间
- 旗形本身也呈倾斜状（通常略微向下倾斜，即使在上涨通道中）

入场方式：
- 止损单入场：在旗形上边界（阻力位）上方1跳动处设置买入止损单
- 限价单入场：如果旗形回撤较深，可在旗形下边界（支撑位）附近用限价单买进

旗形突破的可靠性：
- 旗形越紧凑，突破后的推进越有力
- 旗形持续时间在5~15根K线时效果最佳（不依赖成交量）

三、逆势结构仅诊断（本 Agent **禁止逆势做空**）

上涨通道在 Brooks 体系中常被视为空头旗形，但 **PA Agent 不在通道顶、双顶、MTR、趋势线向下突破等位置产出做空三价**。

仅作结构识别时写入：
- `detected_patterns`：`reversal_attempt`、`double_top_bottom`、`mtr`（四组件齐全时）等
- `watch_points`：监测向下突破测试、极点测试失败、通道演变
- `order_type=不下单`；`terminal.outcome=wait`

若 direction 仍 bullish / AIL：继续评估**顺势做多**（High 1/High 2、§9.0P、假突破失败后的做多侧二次入场）。方向翻转须有 §2.3 重判依据，且 MTR 仍只诊断、不下逆势单。

四、止损单入场与限价单入场的区别

止损单入场（Stop Order Entry）：
- 在信号棒的高点上方 1 跳动处设置买入止损单（本 Agent 仅做多侧）
- 优点：确保入场，不会错过行情
- 缺点：可能被假突破扫损
- 适用场景：信号清晰、市场动能强、通道类型适合突破交易

限价单入场（Limit Order Entry）：
- 在前一棒的低点附近挂买进限价单
- 优点：获得更好的入场价格，盈亏比更高
- 缺点：可能永远不会成交，错过行情
- 适用场景：通道内回撤交易、窄通道中

Al Brooks 的具体建议（做多侧）：
- 买进限价单设在前一棒低点（或略低于前一棒低点）
- 在窄通道中，限价单效果优于止损单
- 在宽通道中，两种方式均可，根据信号强度选择

五、微型通道的交易方法

微型通道（2~10棒，几乎无回撤）的交易原则：

1. 第一个回撤是买入机会
- 微型通道结束后，第一个回撤（通常在第3~5棒内出现）是顺势买进的良机
- 不要在微型通道内部试图做空——程序交易会碾压逆势者

2. 突破微型通道的尝试通常失败
- 如果价格试图向下突破微型通道（即出现低于前低的K线），第一个突破尝试通常会失败
- 交易策略：在突破失败后，顺势买进

3. 特别紧凑的微型通道（连续3棒以上无回撤）
- 行为类似于尖峰（Spike）
- 尖峰结束后通常会有两段回撤：第一段至尖峰中点，第二段至尖峰起点
- 交易策略：等待第二段回撤再入场，胜率更高

六、窄通道vs宽通道的不同交易方法

窄通道交易方法：
- 只做顺势交易（买进）
- 只有计划明确要求等待前一棒低点/回撤位时才使用 pullback 限价；需要确认时使用 breakout，已确认且未偏离时可评估 immediate
- **若有合格信号棒**：可声明 breakout，在信号棒低点下方1跳动触发；最终由 §11 执行解析校验
- 结构止损：最近波段低点或信号棒极点外1跳动
- 利润目标：通道线附近（TP1）+ MM/对侧候选（TP2）

宽通道/台阶交易方法：
- **只做多**（顺 direction / AIL）；禁止通道顶逆势做空
- 顺势交易：High 1/High 2模式，均线回撤买进
- 通道顶双顶/MTR/向下突破 → 仅诊断 + watch_points
- 突破后等回撤确认再入场（突破测试，顺势做多侧）

七、止损设置详解

1. 信号棒止损法（最常用）
- 买进：止损设在信号棒低点下方 1 跳动
- 优点：简单直接，基于价格波动逻辑
- 缺点：如果信号棒实体很大，止损距离过宽

2. 跟踪止损法（Trailing Stop）【Brooks 背景知识；本 Agent 不输出移止损/跟踪止损指令】
- 初始止损按信号棒法设置
- 当价格向有利方向移动超过1倍通道宽度后，将止损移至盈亏平衡点
- 随后每上涨0.5倍通道宽度，将止损上移一次
- 在窄通道中，跟踪止损应更积极

3. ATR动态止损法
- 用平均真实波幅（ATR）的倍数评估止损距离是否过大
- 买进风险参考：入场价 - 1.5×ATR
- 适用于波动较大的市场；但 ATR 位置不能替代信号棒、波段极点或通道边界等图表结构止损

4. 时间止损法【Brooks 背景知识；本 Agent 用 invalidation_condition 表达结构失效，不输出时间平仓指令】
- 如果入场后5~10根K线内价格没有向有利方向移动，平仓出局
- 适用于窄通道中的限价单交易

八、利润目标与平仓策略

1. 测量运动目标（Measured Move Target）
- 从通道起点到当前位置的高度，投射至突破方向
- 顺势做多：从入场点向上投射等于通道高度的距离（作 TP2 候选）

2. 通道高度目标
- 做多目标：当前价格 + 通道高度

3. 分阶段平仓策略【Brooks 背景知识；本 Agent 用 TP1（近端）+ TP2（远端）一次性表达，禁止分批出场】
- 顺势做多：TP1=通道中线候选，TP2=通道线/MM 候选

4. 时间平仓【背景知识；不适用本 Agent】

九、通道突破后的交易策略

1. 顺势突破（向上突破通道线）
- 大多数向上突破会在5根K线内失败
- 交易策略：不要追突破，等待回撤至突破点附近再评估
- 如果回撤幅度很小且迅速恢复上涨，可视为有效突破

2. 逆势突破（向下突破趋势线）【Brooks 背景；本 Agent 禁止逆势做空】
- 识别向下突破与回撤测试背景，用于 `reversal_attempt` / `watch_points`
- **禁止**产出做空三价；若 direction 仍 bullish → wait

3. 突破失败（Fakeout）
- 价格突破后迅速回到通道内
- Brooks 背景：假突破常预示通道延续（非通道结束）
- **PA Agent 交易策略**：在价格回到通道内后，评估**顺主趋势**的二次入场（做多侧：向上假突破失败后做多；做空侧仅诊断）
- 例（direction=bullish）：向下假突破通道下沿失败后 → 评估 Low 2 / §9.0P 做多；**禁止**因假突破反手做空

十、上涨通道演变为交易区间时的应对

识别演变信号：
- 趋势线被多次刺穿，且回撤幅度越来越大
- HH+HL序列被破坏（出现低于前低的波段低点）
- 通道宽度扩大，斜率减小至接近水平
- 在长程结构窗口上，价格进入横盘状态

应对策略：
- 停止使用通道交易策略，切换至震荡区间策略
- **仅顺 direction 一侧**在边界等待（与 `震荡区间交易策略.txt` 一致）；**禁止**双边低买高卖
- 原趋势线可能成为交易区间的中线或阻力位
- 市场通常会回调至通道起点（缺口测试）

【分析流程】

当你在上涨通道中制定交易策略时，请按以下步骤进行：

第一步：评估通道状态
- 当前是窄通道、宽通道还是微型通道？
- 通道处于演变的哪个阶段？
- 是否需要重画趋势线？

第二步：确定交易方向
- **只做多**（顺 direction / AIL）
- 通道顶、双顶、MTR、向下突破 → 仅诊断，不下单

第三步：选择入场模式
- 顺势：High 1/High 2 还是旗形突破？
- 选择止损单入场还是限价单入场？

第四步：确定入场价位
- 具体在什么价格入场？
- 如果使用限价单，挂单价格是多少？
- 如果使用止损单，触发价格是多少？

第五步：计算止损
- 止损设在哪里？
- 止损距离是否相对通道宽度/ATR 过大？

第六步：计算利润目标
- TP1（近端结构位）与 TP2（远端结构位，如 MM、通道对边）
- 须 §10.3 交易者方程通过（RR≥1.0，基于 TP1；高 RR 需 high_rr_review 通过）

第七步：制定应变计划
- 若结构失效：写入 `invalidation_condition` 与 `watch_points`（非持仓操作）
- 若通道突破/转换：建议用户重新跑分析，不在 JSON 中写持仓调整

第八步：输出单次方案
- 在 decision 中给出 order_type + entry/stop/TP1/TP2
- 禁止设置部分平仓单或跟踪止损

【实战规则】

1. 顺势交易规则
- 只在回撤中买进，不追突破
- High 1失败后出现High 2，High 2的胜率通常高于High 1
- 均线（EMA 20）是朋友：回撤至均线附近才考虑买进
- 均线（EMA 20）是动态支撑：回撤至均线附近且出现合格多头信号棒时，**可评估**顺势做多；每笔仍须 §9/§10.3/§14，**禁止**因「每根低点都像机会」而过度给单

2. ~~逆势交易规则~~（已废止）
- 通道顶、双顶、MTR → `order_type=不下单`；仅 watch_points

3. 止损规则
- 信号棒止损法是最低标准
- 如果止损距离超过通道宽度的1.5倍，放弃这笔交易
- 本 Agent 不输出跟踪止损或人为移止损指令

4. 利润目标规则
- 有下单时 TP1 与 TP2 均必填（见文件17、文件23）
- 禁止在 decision 中写「移止损到盈亏平衡」或分批了结

5. 通道突破规则
- 第一个突破尝试通常失败（无论方向）
- 等待突破后的回撤确认（突破测试）再行动
- 如果突破失败且价格回到通道内，评估顺势二次入场，禁止写「加仓」

6. 频率规则
- 若连续两笔方案结构失效，暂停给新方案，在 watch_points 说明需重判 cycle_position

【例外与陷阱】

1. 假信号棒陷阱
- 不是每一根看似反转的K线都是有效的信号棒
- 内包线（Inside Bar）作为信号棒的效果较差
- 信号棒必须伴随后续的确认K线（除非使用止损单入场）

2. 过度交易陷阱
- 在窄通道中，每一根K线都看似买入机会，但不要过度交易
- 同一方向连续多笔交易会迅速累积亏损

3. 移动止损过早陷阱【背景知识；本 Agent 不管理持仓】

4. 忽视空头旗形含义的陷阱
- 上涨通道 = 空头旗形（Brooks 背景）：最终向下突破概率可能高于向上突破
- **PA Agent**：只评估顺势做多；向下突破仅 `reversal_attempt` 诊断 + watch_points，**禁止做空三价**
- 通道后期追多须降 trade_confidence，警惕猛烈回调

5. 突破确认延迟陷阱
- 顺势侧：watch_points 或 §9.0P 限价等待结构确认；禁止「小仓位试探加仓」

6. 单周期多尺度冲突陷阱
- 长程背景与近期方向冲突时，以**近期结构**为交易主方向，长程写入 `htf_context` 作风险提示（见市场诊断框架 场景5/6）
- **禁止**因长程看涨而在近期空头尖峰中做多，或反之
